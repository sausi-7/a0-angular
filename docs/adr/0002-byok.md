# ADR-0002: BYOK (Bring Your Own Key) as the default LLM access model

- Status: Accepted
- Date: 2026-04-17

## Context

Vibe-coding platforms are expensive to run because every generation costs LLM tokens. For an open-source project, absorbing that cost is not viable. We also want contributors to be able to run the platform end-to-end without asking anyone for credentials.

## Decision

Users (and contributors) supply their own API keys for Anthropic, OpenAI, or OpenRouter. The platform stores keys encrypted at rest using a per-deployment `A0_SECRET_KEY`. No platform-owned LLM billing exists in v1.

We use `litellm` as a fallback router so a user can set a single provider or configure provider preference + fallback.

## Consequences

Positive:
- Zero LLM spend for maintainers.
- Contributors with any provider key can run the full loop.
- No business logic around billing, quotas, or shared-key abuse needed in v1.

Negative:
- Onboarding friction: users without any LLM key cannot use the product. Mitigation: add OpenRouter as a single-sign-up path that fronts many providers; document Ollama for fully local use (Phase 1.5).
- Key rotation and revocation require the user to update their settings. Acceptable.

Follow-ups:
- Phase 3 adds per-user rate limits so one user cannot saturate a self-hosted instance.
- A future hosted offering may layer platform-owned billing on top — but that lives outside this repo under a separate commercial entity.

## Alternatives considered

- **Platform-owned API key.** Simplest UX but unbounded cost and abuse risk. Rejected.
- **Free tier via a grant/sponsor.** Worth pursuing later, but dependency on a third party is fragile.
- **Local-only (Ollama) from day one.** Code quality is still far from cloud models. Left as an optional adapter.
